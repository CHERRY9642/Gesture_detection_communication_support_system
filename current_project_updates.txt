SIGN LANGUAGE & GESTURE RECOGNITION SYSTEM

================================================================================
1. PROJECT OVERVIEW
================================================================================
Sign Language & Gesture Recognition is a real-time computer vision system designed for detecting and classifying hand gestures from a live webcam feed. The system utilizes MediaPipe for extracting hand landmarks and TensorFlow/Keras for training and deploying a Multi-Layer Perceptron (MLP) classifier. It supports an extensible vocabulary of signs, enabling applications in accessibility, communication, and human-computer interaction. The project follows a structured workflow from data collection and landmark extraction to model training, TFLite deployment, and a web-based sentence-building application, facilitating non-verbal communication.

1. IMPLEMENTATION AND CODING
--------------------------------------------------------------------------------
The system implementation is modular, divided into Data Processing, Model Training, and Web Application layers.

A. Core Logic (Python & OpenCV):
   - **Landmark Extraction (`extract_landmarks.py`)**: 
     Instead of processing raw images (which is computationally expensive), we use Google's MediaPipe Hands to detect 21 hand landmarks (wrist, thumb, fingers).
     *Input:* Webcam Image (1280x720 RGB)
     *Process:* MediaPipe Inference -> Get (x, y, z) coordinates.
     *Output:* A flat vector of 42 floating-point numbers (21 points × 2 dims).

   - **Feature Engineering**:
     We normalize all points relative to the **Wrist (Point 0)** to ensure the gesture is recognized regardless of where the hand is on the screen.
     *Logic:* `x_normalized = x_point - x_wrist`, `y_normalized = y_point - y_wrist`.

   - **Model Training (`train_mlp.py`)**:
     We use a Multi-Layer Perceptron (MLP) built with TensorFlow/Keras.
     *Architecture:*
     - Input Layer: 42 neurons (features).
     - Hidden Layers: 2 Dense layers with ReLU activation (128 units, 64 units) + Dropout (0.2) to prevent overfitting.
     - Output Layer: Softmax layer with N units (one for each gesture class e.g., 'Hello', 'Doctor').
     *Optimizer:* Adam with sparse_categorical_crossentropy loss.

B. Web Application (Full-Stack):
   - **Backend (`server.py`)**: Built with Flask. It handles routing, user authentication (Login/Signup), and logs predictions to CSV for the dashboard.
   - **Frontend (`app.js` / `script.js`)**: Manages the webcam feed. It captures the video frame, sends it (or the extracted landmarks) to the server/model, and updates the UI with the prediction in real-time.

--------------------------------------------------------------------------------

2. TESTING
--------------------------------------------------------------------------------
We employed a multi-level testing strategy to ensure reliability.

A. Unit Testing (Component Level):
   - **Camera Test**: Verified that OpenCV can access the webcam and release it correctly without crashing.
   - **Data Integrity Test**: Checked `dataset/` folder to ensure no empty folders or corrupt images exist before training.

B. Model Evaluation (Performance Testing):
   - **Confusion Matrix**: Generated to see which gestures get confused (e.g., "Peace" vs "Two"). Adjusted training data for confused classes to improve separation.
   - **Confidence Threshold Testing**: We tested different confidence cutoffs.
     *Result:* Set threshold at **0.75 (75%)**. Predictions below this are flagged as "Low Confidence" or "Unknown" to prevent false positives.

C. Systems Testing (Integration):
   - **Latency/FPS Test**: Measured the time from "Frame Capture" to "Prediction Display".
     *Result:* Average latency < 30ms (approx 33+ FPS) on standard CPU, ensuring smooth real-time performance.
   - **Workflow Test**: Verified the full loop: User Login -> Live Inference -> Gesture Recognition -> Sentence Formation -> Logout.

--------------------------------------------------------------------------------

3. DEPLOYMENT
--------------------------------------------------------------------------------
The project is built for portability and ease of access.

A. Model Optimization:
   - We converted the Keras model (`.h5`) to **TensorFlow Lite (`.tflite`)**.
   - *Benefit:* Reduces model size from ~2MB to ~20KB and speeds up inference by 4x, making it suitable for edge devices or mobile browsers.

B. Local Web Server:
   - Deployed using **Flask** (Python micro-framework).
   - Serves the HTML templates and static assets.
   - Exposes REST APIs (`/api/log_prediction`, `/api/login`) for the frontend to communicate with.

C. User Interface:
   - **Responsive Design:** Using CSS Grid/Flexbox (`style.css`), the interface adapts to Laptops, Tablets, and Mobiles.
   - **Dark Mode:** Implemented a system-wide theme toggle for accessibility.
