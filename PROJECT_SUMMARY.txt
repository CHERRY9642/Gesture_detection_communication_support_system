# GESTURE AI: PROJECT SUMMARY & TECHNICAL SPECIFICATION

## 1. PROJECT OVERVIEW
Gesture AI is a real-time sign language gesture recognition system designed to assist communication. It bridges the gap between sign language users and others by translating hand gestures into text and speech in real-time.

## 2. FRONTEND ARCHITECTURE (app1)
The frontend is a modern React application built for performance and professional aesthetics.

- **Framework**: React 18 with Vite for ultra-fast development and build.
- **Styling**: Pure CSS (Indigo/Slate premium theme) with dark/light mode support.
- **Routing**: `react-router-dom` for seamless navigation between Live Inference and Analytics.
- **State Management**: React Hooks (`useState`, `useEffect`, `useRef`, `useCallback`).

### Key Frontend Features:
- **Live Inference Dashboard**: High-performance camera feed with real-time landmark overlay.
- **Analytics Center**: Data-driven insights powered by `chart.js` and `react-chartjs-2`.
- **Gesture Vocabulary Sidebar**: Quick reference for supported signs.
- **Responsive Layout**: Designed for both desktop and mobile viewing.

---

## 3. MACHINE LEARNING PIPELINE
The core "brain" of the application runs entirely in the browser for maximum privacy and zero latency.

### Step 1: Landmark Extraction (MediaPipe Holistic)
- Uses the `Mediapipe Holistic` model loaded via CDN.
- Detects Hand Landmarks (21 points per hand).
- Maps the skeleton structure onto the UI in real-time (Cyan default, Red when capturing).

### Step 2: Preprocessing
- Landmarks are converted into a **42-D feature vector** (normalized X, Y coordinates).
- Data is normalized relative to the wrist (landmark 0) to ensure distance and hand size don't affect accuracy.
- Maximum value normalization is applied to scale coordinates between -1 and 1.

### Step 3: Inference (TFLite + TensorFlow.js)
- **Model**: Multilayer Perceptron (MLP) exported as a `.tflite` file (approx. 36 KB).
- **Execution**: `tfjs-tflite` engine runs the model locally in the browser.
- **Processing**: Softmax output provides class probabilities (confidence scores).

---

## 4. DETAILED FEATURE BREAKDOWN

### Real-Time Inference:
- Continuous hand tracking with zero-lag prediction updates.
- Visual feedback via hand skeleton drawing (`drawConnectors`, `drawLandmarks`).
- Confidence bar visualization (Green >75%, Yellow >50%, Red <50%).

### Gesture Capture & Sentence Formation:
- **Capture Mode**: A 5-second countdown allows the user to stabilize their hand before the gesture is logged.
- **Sequence Building**: Users can log multiple gestures (e.g., "Where", "Is", "Doctor").
- **Smart Formatting**: A rule-based engine converts sequences into coherent sentences (e.g., "Where is the doctor?").
- **Voice Synthesis**: Integrated Browser `speechSynthesis` API for text-to-speech output.

### Advanced Analytics:
- **KPI Metrics**: Total Predictions, Top Gesture, Average Confidence, and Model F1 Score.
- **Interactive Charts**:
    - **Confidence Trends**: Line chart showing model performance over time.
    - **Gesture Distribution**: Doughnut chart showing the most used signs.
    - **Class Accuracy**: Bar chart showing per-class model precision.
- **Risk Analysis**: High-risk table identifying low-confidence predictions for system audit.

## 5. TECHNICAL SPECIFICATIONS
- **Gesture Classes**: 22 unique hand signs.
- **Model File**: `keypoint_classifier.tflite`
- **Frontend Port**: 5173 (Vite default)
- **Backend API**: Optional Flask integration (port 5000) for persistent logging and advanced sentence logic.
- **Browser Compatibility**: Chrome/Edge (recommended for MediaPipe and Camera stability).
